CoronaDelegate

Type Objective-C Protocol
Revision Current Public Release (2013.2076)
Keywords iOS, enterprise
See also [CoronaRuntime][native.enterprise.ios.CoronaRuntime]

Overview

CoronaDelegate is an Objective-C protocol that gets invoked by the Corona engine at key points in your app's execution, e.g. startup.

Your code must implement this protocol if you wish to add your own functions at the Lua layer.

Methods

(Inherits methods from UIApplicationDelegate)

willLoadMain:

Called right before main.lua is executed. You should register your Lua extensions here.

- (void)willLoadMain:id<Runtime>runtime

runtime. An object that conforms to the [CoronaRuntime][native.enterprise.ios.CoronaRuntime] protocol.

didLoadMain:

Called right after main.lua is executed.

- (void)didLoadMain:id<Runtime>runtime

runtime. An object that conforms to the [CoronaRuntime][native.enterprise.ios.CoronaRuntime] protocol.

Application Lifecycle

You can intercept UIApplicationDelegate events via your implementation of the [CoronaDelegate][native.enterprise.ios.CoronaDelegate] protocol.

This protocol conforms to the UIApplicationDelegate protocol. Corona's internal delegate will call your protocol's method if it is implemented.

Please keep in mind the following:

  • Methods that Apple has deprecated will be ignored.
  • In most cases, your class' version will be invoked after Corona's corresponding version of the UIApplicationDelegate method. There is one situation in which your version will be called before.
  • In situations where the app is about to suspend or go to the background, your method will be called before Corona's version, e.g. applicationWillResignActive: and applicationDidEnterBackground:.
