Facebook Portal Setup

This guide explains how to integrate your app with Facebook using the Facebook Developer Portal for both iOS and Android.

  • Developer Registration
  • Registering an App
  • App Integration
  • Implementing Facebook

Developer Registration

If you haven't already done so, sign in to the Facebook Developer Portal and register as a developer. This is a simple process and it doesn't involve downloading or installing any software.

Registering an App

You must register your app with Facebook to receive a unique App ID. You can do this even if your app isn't yet complete or in the marketplace. In the upper-right area of the apps page, click on the Create New App button. A window will appear; fill in the necessary information:

  • App Name — this should be unique and descriptive of your app.

  • App Namespace — this is optional and isn't required by Corona. If you wish, you can enter a unique code name or identifier for the app.

Next, you'll be asked to answer a security captcha. Do so and continue. In the next screen, you will be presented with your App ID and App Secret.

App Integration

Now look further down on the same page for a series of checkboxes under Select how your app integrates with Facebook.

Important

We recommend that you follow each step carefully, as one misstep can cause Facebook integration to fail. Even if you plan to ignore a certain feature or operating system in your app, we suggest that you read and complete this entire section.

Native iOS App

  • Bundle ID must match the App ID Suffix that you use to build your app. This can be retrieved in the App IDs submenu of the iOS Provisioning Portal where you manage your certificates, devices, provisioning, etc. Here, use the same reverse-domain format, for example com.mycompany.myapp. Do not attempt to use a wildcard App ID.

  • iPhone App Store ID and/or iPad App Store ID is required for Facebook Single Sign-On. See the Build Settings (iOS) section below for further details. You will receive this ID when you add your app to the iTunes Connect portal, even prior to uploading the binary. For more information about adding an app to iTunes Connect, please view this video. If you haven't yet configured the app within the iTunes store, you can use any existing app ID — just don't forget to change it to your App Store ID before you submit the binary to Apple.

  • Facebook Login should be checked as Enabled.

  • Deep Linking should be checked as Enabled. This allows Facebook to use URL schemes to perform specific actions when your app opens. For instance, when the Facebook mobile app launches your app from a link, you will get a launchArgs parameter in your Corona app, from which you can parse an id string.

Native Android App

  • Package Name — use the same package name that you've entered into Corona's build window.

  • Class Name is used for native linking — that is, if someone taps a link to your app in Facebook, it will automatically launch the app. This field must be set to com.ansca.corona.CoronaActivity. Facebook on Android will then launch your app via an Android "intent" using the package name and class name that you've entered on the developer page. The package name (above) is your app's unique string ID and it tells the Android system which app to launch the activity from. As such, there won't be a conflict if more than one Corona app is installed on the same device.

  • Key Hash is required to integrate Facebook Single Sign-On in your Android app. Instructions on how to generate this key hash can be found under Section 4 of this guide.

  • Facebook Login should be checked as Enabled.

  • Deep Linking should be checked as Enabled. This allows Facebook to use URL schemes to perform specific actions when your app opens.  For instance, when the Facebook mobile app launches your app from a link, you will get a launchArgs parameter in your Corona app, from which you can parse an id string.

Implementing Facebook

Assuming everything is set up as indicated, please proceed to the Implementing Facebook guide which discusses how to implement common Facebook tasks and processes in your app.
