Effects

In Corona, you can apply visual effects to display objects with unprecendented ease.

Effects are set by setting the effect parameter of a fill or stroke to the name of the effect. The name is a combination of the type of effect and the name. For example, blur is a filter, so here's how you would apply it to an image:

local bkgd = display.newImage( "bkgd.png" ) bkgd.fill.effect = "filter.blur" There are 3 types of effects available to you:

  • Generators
  • Filters
  • Composites

These effects are categorized based on the number of textures required for them to operate correctly. These are described in more detail below.

Generator Effects

Generators are procedurally-generated effects. They don't operate on any textures/images. Instead, they generate textures in the fragment shader.

"composite.normalMapWith1DirLight"

  • dirLightColor : A vec4 containing RGBA values from 0 to 1. default = { 1, 1, 1, 1 }, min = { 0, 0, 0, 0 }, max = { 1, 1, 1, 1 },

  • dirLightDirection : A vec3 containing the light direction in texture-space. The origin and the growth direction of each axes is the same as described for "pointLightPos" in "composite.normalMapWith1PointLight". We expect this direction vector to be the orientation from where the light is coming from. Not the direction in which it's going. There's no need to normalize this parameter, it's already done internally. default = { 1, 0, 0 }, min = { 0, 0, 0 }, max = { 1, 1, 1 },

  • ambientLightIntensity : A 0 (dark) to 1 (full brightness) float containing the brightness of the ambient light. default = 0, min = 0, max = 1,

NOT supported on older Android devices. See "Android" below.

"composite.normalMapWith1PointLight"

  • pointLightColor : A vec4 containing RGBA values from 0 to 1. default = { 1, 1, 1, 1 }, min = { 0, 0, 0, 0 }, max = { 1, 1, 1, 1 },

  • pointLightPos : A vec3 containing the light position in textures-space. In other words, this is the position where the light is coming from. The XYZ origin is the upper left corner (0,0,0). Positive X grow to the right, positive Y grows towards the bottom, and positive Z grows out of the screen (towards us, the viewer). The XY for the bottom right corder is (1,1). The Z value is the distance from the surface. 0 means the light is flush with the surface. Larger values means it's farther away from the surface. There's no maximum value for Z, but commonly, the user will keep the value between 0 and 1. default = { 1, 0, 0 }, min = { 0, 0, 0 }, max = { 1, 1, 1 },

  • ambientLightIntensity : A 0 (dark) to 1 (full brightness) float containing the brightness of the ambient light. default = 0, min = 0, max = 1,

  • attenuationFactors : A vec3 containing the diffuse light's distance attenuation factors. XYZ are the constant, linear, and quadratic attenuation factors. Read these for a better understanding of the attenuation factors:

https://developer.valvesoftware.com/wiki/Constant-Linear-Quadratic_Falloff http://www.felixgers.de/teaching/jogl/lightAttenuation.html

Note: Distance attenuation DOESN'T affect ambient light intensity.

default = { 0.4, 3, 20 }, min = { 0, 0, 0 }, max = { none, none, none },

NOT supported on older Android devices. See "Android" below.

Android

Certain effects have problems on older Android devices. Most commonly, devices made before 2013, and devices using the ARM Mali 400 GPU). For example, Google Nexus 7, Samsung Galaxy S3, and Amazon Kindle Fire.

To determine if the current device might have problems, use:

if system.getInfo( "gpuSupportsHighPrecisionFragmentShaders" ) then
    -- This device is ok.
    -- This device WON'T have problems.
else
    -- This device ISN'T ok.
    -- This device WILL have problems.
end
