Overview

gamestick.IAPPurchaseItem() makes a request to process a purchase through the GameStick servers. Note: On confirmation that a purchase has been successful, developers will need to implement their own code to determine what this purchase will do.

Syntax

gamestick.IAPPurchaseItem( {itemID = "string", requestID = 1}, listener)
options (required)

Table. A table of parameters.

listener (required)

TYPE. The listener function which will handle the API response.

Format for options

The options table contains the following properties:

itemID (required)

TYPE. A string representing the ID number for the item you wish to get the URL for. The ID number can be retrieved using gamestick.IAPGetItemsForPurchase() if not already known.

requestID (required)

TYPE. An integer which will be returned with the response data as a property called "id". This is used to simplify matching up requests and responses when multiple requests are made close together

Examples

local function gamestickListener( event )

    --Failed to get a valid response from server
    if event.isError then
        native.showAlert( "Error: "..event.name, event.message, { "OK" } )
    else
        if event.name == "IAPPurchaseItem" then
            --NOTE: BOTH OF THE RETURNED FIELDS ARE EXTREMELY IMPORTANT IN DETERMINING WHETHER
            --A PURCHASE HAS BEEN SUCCESSFUL, AND WHICH ITEM WAS BOUGHT.

            --event.id will match the requestID from the function call. If 2 purchases are made in quick succession
            --this id will allow you to determine which purchase you are receiving a response for.  
            local id  = event.id

             --event.response is a boolean stating whether or not the purchase was successful.
             --If this is false, the purchase was not successful so this will need to be conveyed to the user.
            local response = event.response
        end
    end
end

gamestick.IAPPurchaseItem( {itemID = "string", requestID = 1}, gamestickListener)
