fortumo.makePayment()

Revision Current Public Release (2013.2076)

Sample code

Overview

Starts the payment process.

Syntax

fortumo.makePayment(REQUEST, CALLBACK)
fortumo.makePayment(REQUEST)
REQUEST (required)

Table. A payment request.

CALLBACK (optional)

function. A callback which will be triggered with the status of the provided payment request upon completion.

Examples

function onPaymentComplete(response)
    -- Indicates the transaction status. 0 - cancelled, 1 - pending (waiting SMS), 2 - billed, 3 - failed, 4 - use alternative method
    response.billingStatus

    -- Amount of credits paid for by the user (if the user buys "15 crystals" then this value is "15")
    response.creditAmount

    -- Name of the credits paid for by the user (in case of "15 crystals" this would be "crystals")
    response.creditName

    -- The message id of the transaction
    response.messageId

    -- Payment code specific to the end-user's device
    response.paymentCode

    -- Amount paid by the user in a fractional format (X.YZ)
    response.priceAmount

    -- ISO 4217 currency code (like USD, EUR...)
    response.priceCurrency

    -- Used to identify non-consumable payments (like level_6)
    response.productName

    -- Service ID this message was sent for
    response.serviceId

    -- The user ID
    response.userId
end

local request = fortumo.newPaymentRequest();

request:setService("serviceId", "appSecret");
request:setProductName("productName");
request:setConsumable(false);

fortumo.makePayment(request, onPaymentComplete)
