inmobi.initWithParams()

Type function
Library inmobi.*
Return value boolean
Revision Current Public Release (2013.2076)
Keywords
Sample code
See also

Overview

Syntax

inmobi.initWithParams( appId [, listener] [, optionalParams] )
appId (required)

String. This is a mandatory param before making any ad request. You can get the appId from www.inmobi.com

listener (optional)

Function. This function is used to listen to the different inmobi banner/interstitial events.

optionalParams (optional)

Table. The optional parameters will help inmobi to fetch more relavent ads.

Format for optionalParams

The optionalParams table contains the following properties:

logLevel (optional)

Constant. This will print the InMobi logs. Following are the valid values: inmobi.CONSTANTS.LOGLEVEL_NONE inmobi.CONSTANTS.LOGLEVEL_DEBUG inmobi.CONSTANTS.LOGLEVEL_VERBOSE

gender (optional)

Constant. Following are the valid values: inmobi.CONSTANTS.GENDER_MALE inmobi.CONSTANTS.GENDER_FEMALE inmobi.CONSTANTS.UNKNOWN

education (optional)

Constant. This will print the InMobi logs. Following are the valid values: Inmobi.CONSTANTS.EDUCATION_HIGHSCHOOLORLESS Inmobi.CONSTANTS.EDUCATION_COLLEGEORGRADUATE Inmobi.CONSTANTS.EDUCATION_POSTGRADUATEORABOVE Inmobi.CONSTANTS.EDUCATION_UNKNOWN

ethnicity (optional)

Constant. This will print the InMobi logs. Following are the valid values: Inmobi.CONSTANTS.ETHNICITY_ASIAN Inmobi.CONSTANTS.ETHNICITY_HISPANIC Inmobi.CONSTANTS.ETHNICITY_AFRICAN_AMERICAN Inmobi.CONSTANTS.ETHNICITY_CAUCASIAN Inmobi.CONSTANTS.ETHNICITY_OTHER Inmobi.CONSTANTS.ETHNICITY_UNKNOWN

maritialStatus (optional)

Constant. This will print the InMobi logs. Following are the valid values: Inmobi.CONSTANTS.MARITAL_STATUS_SINGLE Inmobi.CONSTANTS.MARITAL_STATUS_DIVORCED Inmobi.CONSTANTS.MARITAL_STATUS_ENGAGED Inmobi.CONSTANTS.MARITAL_STATUS_RELATIONSHIP Inmobi.CONSTANTS.MARITAL_STATUS_UNKNOWN

hasChildren (optional)

Constant. This will print the InMobi logs. Following are the valid values: Inmobi.CONSTANTS.TRUE Inmobi.CONSTANTS.FALSE Inmobi.CONSTANTS.UNKNOWN

sexualOrientation (optional)

Constant. This will print the InMobi logs. Following are the valid values: Inmobi.CONSTANTS.SEXUAL_ORIENTATION_STRAIGHT Inmobi.CONSTANTS.SEXUAL_ORIENTATION_BISEXUAL Inmobi.CONSTANTS.SEXUAL_ORIENTATION_GAY Inmobi.CONSTANTS.SEXUAL_ORIENTATION_UNKNOWN

dob (optional)

String. Format: dd-mm-yyyy

income (optional)

Number. Numeric (should be in USD)

age (optional)

Number. Range: (0-100)

language (optional)

String. Eg: eng http://www-01.sil.org/iso639-3/iso-639-3.tab

postalCode (optional)

String.

areaCode (optional)

String.

interests (optional)

String.

postalCode (optional)

String.

Examples


local inmobi = require "plugin.inmobi"

-- You can get the appId from www.inmobi.com
local appId = "__APP_ID__"

local networkParams = {
        logLevel = inmobi.CONSTANTS.LOG_LEVEL_DEBUG,
        gender = inmobi.CONSTANTS.GENDER_MALE,
        education = inmobi.CONSTANTS.EDUCATION_HIGHSCHOOLORLESS,
        ethnicity = inmobi.CONSTANTS.ETHNICITY_HISPANIC,
        dob = "29-11-1984",
        income = "70000",
        age = "20",
        maritalStatus = inmobi.CONSTANTS.MARITAL_STATUS_SINGLE,
        hasChildren = inmobi.CONSTANTS.FALSE,
        sexualOrientation = inmobi.CONSTANTS.SEXUAL_ORIENTATION_STRAIGHT,
        language = "ENG",
        postalCode = "1111",
        areaCode = "465",
        interests = "adventure sports, water sports",
        deviceIdMasks = { inmobi.CONSTANTS.EXCLUDE_VENDOR_ID }
}

inmobi.initWithParams(appId, adListener, networkParams)
