together.*

Type library
Revision Current Public Version 1.1
Keywords together
Sample code github samples
See also http://playstogether.com
Availability Starter, Basic, Pro

Overview

The Together class is the root class in the Together api. It contains essential configuration settings, an object representing the logged in User, and various Managers for creating, deleting, and retrieving various Together objects.

Sign Up

To use the Together service, please sign up for an account.

Platforms

  • Android: Yes
  • iOS: Yes
  • Mac: Yes
  • Win: Yes
  • Kindle: No
  • NOOK: No

Syntax

It is best to assign the GetInstance call to a global either you have created or to the declared global in the library, "g_Together". After signing up you can connect/login by passing your generated keys to the initialization call of the created instance.

local together = require("plugin.together");
g_Together = together:GetInstance();
g_Together:Initialize(yourPublicKey, yourPrivateKey, yourGameKey, yourPlatform);

Callbacks

All network methods that require a callback function will invoke your callback with the same object. Details about this "Event" object can be found here.

Project Settings

SDK

When you build using the Corona Simulator, the server automatically takes care of integrating the plugin into your project.

All you need to do is add an entry into a plugins table of your build.settings. The following is an example of a minimal build.settings file:

settings =
{
    plugins =
    {
        -- key is the name passed to Lua's 'require()'
        ["plugin.together"] =
        {
            -- required
            publisherId = "com.playstogether",
        },
    },      
}

LoginUser()

Logs in the User. This should always be the first Together network message called. If no User GUID is provided, a new user account will be created anonymously and assigned a GUID which can be used to log the user in again later or register them with Facebook or via Custom form.

Note Currently anonymous users will NOT be deleted. However it is possible that a user cleansing system will be implemented in the future that removes unregistered accounts after 3 to 6 months of inactivity.

Returns void

Syntax

Together:LoginUser( callbackFunc )

Parameters

function callbackFunc

The function to call upon completion.
