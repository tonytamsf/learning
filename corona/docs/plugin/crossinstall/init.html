ads.init()

Type library
Library crossinstall.*
Revision Current Public Release (2013.2076)
Keywords CrossInstall
Sample code
See also
Availability Starter, Pro, Enterprise

Overview

ads.init() initializes the Corona ads library by specifying the CrossInstall ad network and the application identifier.

Android

You must add the following permissions to the build.settings file to use ads on Android.

settings =
{
   android =
   {
      usesPermissions =
      {
         "android.permission.INTERNET",
         "android.permission.ACCESS_NETWORK_STATE",
         "android.permission.READ_PHONE_STATE",
      },
   },
}

Syntax

ads.init( providerName, appId [, listener]  )
providerNAme (required)

String. Enter "crossinstall".

appId (required)

String. Your CrossInstall App Identifier (e.g. "kcrC").

listener (optional)

Table. Function that will receive an event indicating if showing an ad has succeeded or failed. This will be called for every new ad that is requested, according to the interval given to the ads.show() function. Provides a property event.isError which will be true if the system failed to retrieve an ad. This can occur if the ad network is unreachable or is out of inventory/ads, in which case nothing is shown on screen. event.isError will be false if an ad was successfully retrieved and displayed on screen.

Example

local ads = require "ads"

local function adListener( event )
  -- event table includes:
  --    event.provider
  --    event.isError (e.g. true/false )
  --    event.response (e.g. "Success"/<errorMessage>/"Fullscreen closed")

  if event.isError then
    -- handle failed ad loading
  else
    -- optionally handle interstitial closing, etc
  end
end

ads.init( "crossinstall", "your crossinstall app id, like AbCd", adListener )
ads.show( "fullscreen", "my placement name", {timeout=10} )
