Overview

Displays the operating system's default popup window for a specified service.

Syntax

native.showPopup( name, params )
name (required)

String. The string name of the popup to be shown. It can be one of the following: addressBook

params (required)

Table. Table of parameters. Each property of the table (except for the option key) is optional. See Format for options below.

Format for params

The params table contains the following properties:

option (required)

String. The string name of the popup to be shown. It can be one of the following: pickContact, viewContact, newContact, unknownContact

"pickContact"

For the "pickContact" popup, the following properties are supported:

  • options.hideDetails is a boolean. If set to true, then the address book will be dismissed upon selecting a contact.
  • options.performDefaultAction is a boolean. If set to true then tapping/touching a contacts phone number or email will perform the default iOS action (such as call contact, show the compose email popup).
  • options.filter is a table. It should contain items (strings) to show in the detail view section of a contact. Eg: filter = { "phone", "email", "birthday" }
  • options.listener is a listener. This is the callback function you want to be called when the popup is dismissed. The onComplete callback function passes back an event table with the following properties: * event.name: String. contact. * event.type: A String. which is either: pickContact, viewContact, newContact, unknownContact, depending on the popup shown. * event.data: A Table. containing information of the selected contact. i.e. firstName, lastName, email, phone.

Supported strings for options.filter are:

`phone`, `email`, `address`
`birthday`, `relatedNames`,
`socialProfiles`, `instantMessagingProfiles`,

"viewContact"

For the "viewContact" popup, the following properties are supported:

  • options.performDefaultAction is a boolean. If set to true then tapping/touching a contacts phone number or email will perform the default iOS action (such as call contact, show the compose email popup).
  • options.isEditable is a boolean. If set to true the user can edit the person’s information.
  • options.name is a string. This is the name of the contact you wish to view.
  • options.filter is a table. It should contain items to show in the detail view section of a contact. e.g.: filter = { "phone", "email", "birthday" }
  • options.listener is a listener. This is the callback function you want to be called when the popup is dismissed. The onComplete callback function passes back an event table with the following properties: * event.name: String. contact. * event.type: A String. which is either: pickContact, viewContact, newContact, unknownContact, depending on the popup shown. * event.data: A Table. containing information of the selected contact. i.e. firstName, lastName, email, phone.

"newContact"

For the "newContact" popup, the following properties are supported:

  • options.data is a table. This is a table containing key/value pairs to pre-fill a new contact's details with. e.g.: data = { firstName = "John", lastName = "Doe", organization = "Corona Labs", homePhone = "939222832", workPhone = "939392832", homeEmail = "jondoe@someorganization.com", workEmail = "johndoe@home.com", }
  • options.listener is a listener. This is the callback function you want to be called when the popup is dismissed. The onComplete callback function passes back an event table with the following properties: * event.name: String. contact. * event.type: A String. which is either: pickContact, viewContact, newContact, unknownContact, depending on the popup shown. * event.data: A Table. containing information of the selected contact. i.e. firstName, lastName, email, phone.

"unknownContact"

For the "unknownContact" popup, the following properties are supported:

  • options.performDefaultAction is a boolean. If set to true then tapping/touching a contacts phone number or email will perform the default iOS action (such as call contact, show the compose email popup).
  • options.allowsActions is a boolean. This specifies whether buttons appear to let the user perform actions such as sharing the contact, initiating a FaceTime call, or sending a text message.
  • options.allowsAdding is a boolean. If set to true the “Add to Existing Contact” or “Create New Contact” buttons will be available on the contacts overview screen.
  • options.alternateName is a string. This provides a string that is displayed instead of the contacts first and last name.
  • options.message is a string. This string is displayed below alternateName.
  • options.data is a table. This is a table containing key/value pairs to amend to a contact or to create a new contact with. e.g.: data = { organization = "Corona Labs", homePhone = "939222832", workPhone = "939392832", workEmail = "fake@coronalabs.com", homeEmail = "fake@home.com", }
  • options.listener is a listener. This is the callback function you want to be called when the popup is dismissed. The onComplete callback function passes back an event table with the following properties: * event.name: String. contact. * event.type: A String. which is either: pickContact, viewContact, newContact, unknownContact, depending on the popup shown. * event.data: A Table. containing information of the selected contact. i.e. firstName, lastName, email, phone.

Examples


-- Function to execute on completion of the address book ( i.e. when it is dismissed )
local function onComplete( event )  
    print( "event.name:", event.name );
    print( "event.type:", event.type );

    -- event.data is either a table or nil depending on the option chosen
    print ( "event.data:", event.data );

    -- If there is event.data print it's key/value pairs
    if event.data then
        print( "event.data: {" );

        for k, v in pairs( event.data ) do
            print( k , ":" , v );
        end 

        print( "}" );
    end
end

-- Pick contact options
local pickContactOptions = 
{
    option = "pickContact",
    -- If true, then the address book will be dismissed upon selecting a contact. 
    hideDetails = true,
    -- If true then tapping/touching a contacts phone number or email will perform the default iOS action (such as call contact, show the compose email popup).
    performDefaultAction = true,

    -- Items to show in the detail view section of a contact.
    filter = 
    {
        "phone", "email",
        "birthday",
    },
    -- Function to execute on completion
    listener = onComplete,
}

native.showPopup( "addressBook", pickContactOptions );
