CompositePaint

Parent Paint
Library display.*
Revision Current Public Release (2013.2076)
Keywords multitextures.
Sample code
See also shape.fill
shape.stroke

Overview

Composite paints contain multiple textures. They enable multitexturing.

Specifying composites

Bitmap paints are specified by Lua tables:

local paint = {
    type = "composite",
    paint1=,
    paint2=
}

local rect = display.newRect( 0, 0, 100, 100 )
rect.fill = paint
rect.fill.effect = "composite.average"
type (required)

String. This must be "composite".

paint1 (required)

Table. A table that specifies a BitmapPaint. See 'Limitations' below.

paint2 (required)

Table. A table that specifies a BitmapPaint. See 'Limitations' below.

baseDir (optional)

Constant. Specifies the base directory where filename is located. Options include system.ResourceDirectory, system.DocumentsDirectory, system.TemporaryDirectory and system.CachesDirectory. Default is system.ResourceDirectory.

Limitations

Because of the way multitexturing works, both paint1 and paint2 will be rendered using the same texture coordinates. Because GradientPaints and ImageSheetPaints use different texture coordinates from plain BitmapPaints, you will get unexpected results unless you use plain BitmapPaints for paint1 and paint2.
