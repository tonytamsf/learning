object:getUserLocation()

Type Function
Object Map
Library native.*
Return value location
Revision Current Public Release (2013.2076)
Keywords getUserLocation
Sample code /CoronaSDK/SampleCode/Interface/MapView
See also

Overview

Returns a table containing values for the user's current location. The fields of this table are the same as those currently used in GPS location events.

If the returned table's location.errorCode field is not nil, then this indicates that the application has failed to fetch the user's current location. This can happen for the following reasons:

  • Current location has not been received yet.
  • Current location tracking is not available on the device.
  • The application does not have permission.

Gotchas

In order to retrieve current location on Android, you must set the following permissions in the "build.settings" file. Otherwise this function will return a location table with the location.errorCode field set and the current location marker will not appear in the map.

settings =
{
    android =
    {
        usesPermissions =
        {
            -- Permission to retrieve current location from the GPS.
            "android.permission.ACCESS_FINE_LOCATION",

            -- Permission to retrieve current location from WiFi or cellular service.
            "android.permission.ACCESS_COARSE_LOCATION",
        },
    },
}

Syntax

object:getUserLocation()

Example

local myMap = native.newMapView( 0, 0, 320, 480 )
local locationTable  = myMap:getUserLocation()
local locationtxt = display.newText("My location is: ", 0, 0, native.systemFont, 16)

if locationTable.errorCode then
    locationtxt.text = locationtxt.text .. locationTable.errorMessage
else
    locationtxt.text = locationtxt.text .. locationTable.latitude .. ", " ..locationTable.longitude
end
