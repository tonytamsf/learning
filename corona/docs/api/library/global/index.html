Overview

The (globals) section covers basic functions of Lua 5.1, which is the underlying programming language of the Corona SDK. All of the functions and objects listed in the Functions section can be called as-is (without a prefix). For example:

globals.rawget()   -- WRONG; will produce an error

rawget()   -- Correct

You can find more information at the official Lua website.

The _G global variable

_G is a global variable (not a function) that holds the global environment (that is, G.G = _G). Lua itself does not use this variable; changing its value does not affect any environment, nor vice-versa. (Use setfenv() to change environments.)
