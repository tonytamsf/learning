Overview

The Corona ads library lets you display ads within your application. The following ad networks are currently supported:

  • inneractive
  • Vungle
  • InMobi
  • AdMob
  • iAds

For more information, please refer to the Implementing Ads guide.

Functions

ads.hide()

ads.init()

ads.show()

Examples

The following code demonstrates basic usage of the ads library with iAds as the provider:

local ads = require "ads"

local function adListener( event )
local msg = event.response
    if event.isError then
        -- Failed to receive an ad, we print the error message returned from the library.
        print(msg)
    end
end

ads.init( "iads", "myAppId", adListener )
ads.show( "banner", { x=0, y=0 } )

And this example illustrates usage of the InMobi network:

-- Import library and initialize with personal application key 
local ads = require "ads"
ads.init( "inmobi", your ad key )

-- Set the ads (updates every 60 seconds)
ads.show( "banner320x48", { x=0, y=100, interval=60, testMode=false } )

Project Settings

To use ad services, you must specify which ad plugin(s) should be included in the compiled build. When you build the app from the Corona Simulator, the server automatically integrates the plugin(s) into your project, but you must first add an entry for each desired ad provider into the plugins table of build.settings.

InMobi

settings =
{
    plugins =
    {
        -- key is the name passed to Lua's 'require()'
        ["CoronaProvider.ads.inmobi"] =
        {
            -- required
            publisherId = "com.coronalabs",
        },
    },      
}
