ads.show()

Type Function
Library ads.*
Return value None
Revision Current Public Release (2013.2076)
Keywords ads, advertising
Sample code /CoronaSDK/SampleCode/Ads/InMobi
/CoronaSDK/SampleCode/Ads/inneractive

See also ads.hide()

Overview

ads.show() begins showing ads at the given screen location on the provided refresh period.

Note: The ad dimensions are in "points" and not pixels. This means that the display size in pixels will be double on a retina/HD device such as the iPhone4.

iOS

The iAds "banner" banner type will automatically scale to iPad/iPhone, and to landscape/portrait.

The iAds provider has known issues on iOS6; the banners begin displaying after some delay from the startup, typically 30-60 seconds.

If you're developing a "universal" app, i.e for both iPhone4 and Retina iPad, ads will not dynamically scale to the iPad. If your config.lua file defines a 480×960 screen, a "480×960" ad will appear proportionally smaller on the iPad compared to the iPhone4. Thus, it's wise to detect the device type and choose the proper ad dimensions.

Android

You must add the following permissions to the build.settings file to use ads on Android.

Note: Android does not support iAds.

settings =
{
   android =
   {
      usesPermissions =
      {
         "android.permission.INTERNET",
         "android.permission.ACCESS_NETWORK_STATE",
         "android.permission.READ_PHONE_STATE",
      },
   },
}

Syntax

ads.show( adUnitType [, params] )
adUnitType (required)

String. String indicating the type of ad to show on screen. See "Ad Unit Values" below.

params (optional)

Table. Specify optional parameters. See "Parameter Table Properties" below.

Ad Unit Values

iAds

iAds supports the following ad types on all iOS devices:

  • "banner"

iAds supports the following ad banner type only on iPad:

  • "interstitial"

inMobi

InMobi supports the following ad types on all devices:

  • "banner320x48"
  • "banner300x250"

InMobi supports the following larger ad banners types only on iPad:

  • "banner728x90"
  • "banner468x60"
  • "banner120x600"

inneractive

inneractive supports the following ad types on all devices:

  • "banner"
  • "text"
  • "fullscreen"

The banner ad type has a height of 53px and 90px for iPhone and iPad respectively.

AdMob

AdMob supports the following ad types on all devices:

  • "banner"
  • "interstitial"

Parameter Table Properties

x (optional)

Number. The upper left banner position. Defaults to 0. For iAds, this applies only to the "banner" banner type.

y (optional)

Number. The top banner position. Defaults to 0. For iAds, this applies only to the "banner" banner type.

interval (optional)

Number. The ad refresh time (in seconds). Defaults to 10 for InMobi. Defaults to 60 for inneractive. Not applicable for iAds.

testMode (optional)

Boolean. Set to true for testing account. Defaults to false. Not applicable for iAds.

Examples

iAds

local ads = require "ads"

local function adListener( event )
local msg = event.response
    if event.isError then
        -- Failed to receive an ad, we print the error message returned from the library.
        print(msg)
    end
end

ads.init( "iads", "myAppId", adListener )
ads.show( "banner", { x=0, y=0 } )

InMobi

local ads = require "ads"

local function adListener( event )
    if event.isError then
        -- Failed to receive an ad.
    end
end

ads.init( "inmobi", "myAppId", adListener )
ads.show( "banner320x48", { x=0, y=100, interval=60, testMode=false } )

inneractive

local ads = require "ads"

local function adListener( event )
    if event.isError then
        -- Failed to receive an ad.
    end
end

ads.init( "inneractive", "myAppId", adListener )
ads.show( "banner", { x=0, y=0, interval=60 } )

AdMob

local ads = require "ads"

local function adListener( event )
    if event.isError then
        -- Failed to receive an ad.
    end
end

ads.init( "admob", "myAppId", adListener )
ads.show( "banner", { x=0, y=0 } )
