display.contentScaleX

Type Number
Library display.*
Revision Current Public Release (2013.2076)
Keywords contentWidth
Sample code /CoronaSDK/SampleCode/
See also

Overview

The ratio between content pixel and screen pixel width.

This value will always be 1 unless your project's config.lua file specifies your content's height and width (see example below). Different scale modes (letterbox, zoomEven, and so forth) will result in different scale factors.

Example

main.lua

local scalex = display.contentScaleX
local scaley = display.contentScaleY

-- display scale values
print("scalex", "scaley", scalex, scaley)

config.lua

This configures the size of the content (the virtual screen that scales to the physical screen size)

application =
{
    content =
    {
        width = 320,
        height = 480,
        scale = "letterbox"
    },
}

Output

The above sample has the following output:

-- iPhone
scale_x scale_y 1       1

-- iPhone 4
scale_x scale_y 0.5     0.5

-- Droid
scale_x scale_y 0.66666668653488        0.66666668653488

-- iPad
scale_x scale_y 0.46875 0.46875
