storyboard.newScene()

Type Function
Library storyboard.*
Return value Table
Revision Current Public Release (2013.2076)
Keywords storyboard, new, scene, newScene
Sample code /CoronaSDK/SampleCode/Interface/Storyboard
See also storyboard.gotoScene()

Overview

Used to create new scene objects to be used with the Storyboard API (see that page for a scene tempalate module).

Syntax

storyboard.newScene( [ sceneName ] )
sceneName (optional)

String. If you are NOT using a separate module for your scene, you must specify a unique scene name here. If you are using a separate module for your scene, you do not need to pass this argument. Instead, the module name will be used. For intance, if scene1.lua is your scene module, then the scene name will automatically be set to "scene1".

Examples

Scene module (recommended)

-- scene1.lua

local storyboard = require "storyboard"
local scene = storyboard.newScene()

function scene:createScene( event )

    -- object creation code here

end
scene:addEventListener( "createScene" )

function scene:enterScene( event )

    -- scene logic goes here
end
scene:addEventListener( "enterScene" )

return scene

Multiple scenes in a single module

This method is not recommended because if a scene does not have its own module, when storyboard.removeAll() or storyboard.removeScene() is called on the scene, it will cease to exist, meaning you'll have to recreate it and re-add all events to it. When a scene has its own module, however, everything is defined in the module so you don't have the same limitation.

-- main.lua

local storyboard = require "storyboard"

local scene1 = storyboard.newScene( "scene1" )
local scene2 = storyboard.newScene( "scene2" )

function scene1:createScene( event )
    -- scene creation code for scene1 here...
end
scene1:addEventListener( "createScene" )

function scene2:createScene( event )
    -- scene creation code for scene2 here...
end
scene2:addEventListener( "createScene" )

-- and so on...
