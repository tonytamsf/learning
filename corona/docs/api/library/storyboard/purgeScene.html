storyboard.purgeScene()

Type Function
Library storyboard.*
Return value none
Revision Current Public Release (2013.2076)
Keywords storyboard, purge, scene, purgeScene
Sample code /CoronaSDK/SampleCode/Interface/Storyboard
See also storyboard.removeScene()
storyboard.purgeAll()
[storyboard.getSceneName]

Overview

Unloads the specified scene's scene.view property, which is a group that contains all of the scene's display objects. It takes one (required) argument, sceneName, which is the name of the scene (usually the name of the module, without the ".lua" extension) you are attempting to unload. If the scene has its own module, this function preserves the scene's module in the global package.loaded table.

When the scene is re-loaded...

When storyboard.gotoScene() or storyboard.loadScene() is later called on the scene specified in this function, the module will already be loaded, but the createScene event will be dispatched to re-create the scene.view property.

NOTE: This function may be automatically called on the LRU (least recently used) scene if a "low memory warning" is received, or it may be triggered explicitly by the user for memory/performance considerations, in cases where the user plans on loading the scene again at some point in the near future. This automatic purging functionality is the default behavior, but it can be disabled by setting the storyboard.disableAutoPurge property to true.

Associated events

Before the scene.view display group is removed (scene purge), a destroyScene event is dispatched to the specified scene. The destroyScene event listener is a common place for final scene "clean-up" logic.

Syntax

storyboard.purgeScene( sceneName )
sceneName (required)

String. This is the name of the scene you want purged. For instance, if you want to purge scene1.lua, you would pass "scene1" as sceneName.

Example

local storyboard = require "storyboard"

-- the following will remove scene1's scene.view display group
storyboard.purgeScene( "scene1" )
