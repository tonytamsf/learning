physics.newJoint()

Type Function
Library physics.*
Return value Joint
Revision Current Public Release (2013.2076)
Keywords physics, joints

Overview

Joints are used to assemble more complex game objects from multiple rigid bodes. To construct a joint, first construct the bodies that will be joined, and then call this method with the desired joint constructor type.

To remove a joint, use joint:removeSelf() or display.remove( joint ).

Gotchas

See the Physics Joints guide to see how to deal with the various joints.

This API should not be used in a collision event handler.

Some joints may be unstable due to the fact that Box2D uses continuous physics. You can turn this off using physics.setContinuous().

Syntax

physics.newJoint( jointType, ... )
jointType (required)

String. One of the following joint types:

  • "pivot"
  • "distance"
  • "piston"
  • "friction"
  • "weld"
  • "wheel"
  • "pulley"
  • "touch"
object1 (required)

DisplayObject. First object used in joint.

object2

DisplayObject. Second object used in joint.

anchorX, anchorY

Numbers. Defines the x and y locations of the joint anchor points.

Distance joints

A distance joint joins two bodies at a fixed distance. The initial arguments are the two bodies joined, followed by an (x,y) anchor point for each body. Anchor points are declared in the world coordinates.

physics.newJoint( "distance", object1, object2, anchorX1, anchorY1, anchorX2, anchorY2 )

Friction joints

A friction joint is a special kind of pivot joint that resists motion, and is therefore "sticky".

physics.newJoint( "friction", object1, object2, anchorX, anchorY )

Piston joints

A piston joint (also known as a “prismatic joint” in Box2D terms) joins two bodies along a single, restricted axis of motion, like the pistons or a shock absorber in a car. The initial arguments are the two bodies joined, followed by an anchor point in the first body and a vector defining the axis along which motion is allowed.

physics.newJoint( "piston", object1, object2, anchorX1, anchorY1, axisDistanceX, axisDistanceY )

Pivot joints

A pivot joint (also known as a “revolute joint” in Box2D terms) joins two bodies at an overlapping point. The initial arguments are the two bodies joined, followed by an anchor point in the global coordinates of the world.

physics.newJoint( "pivot", object1, object2, anchorX, anchorY )

Pulley joints

A pulley joint attaches two bodies with an imaginary rope whose length remains constant: if one body is pulled down, the other one will move up.

This constructor is a bit complicated, because it must specify a joint anchor point within each body, and also a stationary anchor point in world coordinates for each side of the “rope” to hang from. Finally, there is a “ratio” property to optionally simulate a block and tackle arrangement, in which one side of the rope moves faster than the other. By default, this ratio is 1.0, which simulates a simple pulley.

physics.newJoint( "pulley", object1, object2, anchorX1, anchorY1, anchorX2, anchorY2, x1, y1, x2, y2, ratio )

Touch joints

A touch joint (based on the Box2D “mouse joint”) connects a single object to the current position of an onscreen touch. This connection is an elastic joint with specifiable strength and behavior, but since it is impossible to apply infinite forces to the simulation, the dragged object may "lag behind" a rapidly moving touch.

physics.newJoint( "touch", object1, targetX, targetY )

Weld joints

A weld joint “welds” two bodies together at a specified point in world coordinates.

physics.newJoint( "weld", object1, object2, anchorX, anchorY )

Wheel joints

A wheel joint (also known as a “line joint” in Box2D terms) combines a piston and a pivot joint, like a wheel mounted on the shock absorber of a car. Most of its properties are inherited from the standard piston joint; the difference is that the body at the end of the axis is allowed to rotate freely.

physics.newJoint( "wheel", object1, object2, anchorX, anchorY, axisDistanceX, axisDistanceY )

Example

local myJoint = physics.newJoint( "pivot", crateA, crateB, 200,300 )
