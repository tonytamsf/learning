timer.performWithDelay()

Type Function
Library timer.*
Return value Object
Revision Current Public Release (2013.2076)
Keywords timer, delay, performWithDelay
Sample code /CoronaSDK/SampleCode/GettingStarted/Timer
See also timer.cancel()

Overview

Call a specified function after a delay. This function returns an Object that can be used with other timer.* functions.

This function returns a handle that could be passed to timer.cancel() in order to cancel the invocation of listener.

Syntax

timer.performWithDelay( delay, listener [, iterations] )
delay (required)

Number. The delay in milliseconds (e.g. 1000 = 1 second)

listener (required)

Listener. The listener to invoke after the delay. May be either a function listener or a table listener. If a table, it must have a timer method because timer events are sent to the listener.

iterations (optional)

Number. Optionally specifies the number of times listener is to be invoked. By default, it is 1; pass 0 if you want it to loop forever.

Examples

Function Listener

local function listener( event )
    print( "listener called" )
end

timer.performWithDelay( 1000, listener )

Table Listener

local listener = {}
function listener:timer( event )
    print( "listener called" )
end

timer.performWithDelay( 1000, listener )

Passing parameters

To pass a parameter to the listener, use a Lua closure, as in the following example:

local function spawnBall( randomPosition )
    ballPosition = display.newCircle( 100, 100, 20 )
    ballPosition.x = randomPosition
end

-- obtain a random number for the spawned object's x position
local randomPosition = 100 + math.random(200)

-- wrap spawnBall and randomPosition inside a closure
local myClosure = function() return spawnBall( randomPosition ) end
timer.performWithDelay(2000, myClosure, 1)
